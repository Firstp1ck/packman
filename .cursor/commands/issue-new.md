# issue-new

Create a GitHub **issue** description for the problem or idea the user describes.

Keep it: simple, maintainer-friendly (minimal jargon), clear, and concise. Avoid duplicating the title in the body. Write from an **end user** perspective.

## Title format

`[Type]: [Title]`

**Types:** Feature, Bug, Improvement, Doc, Question

**Example:** `[Bug] Search mode does not clear after upgrading from all-updates overlay`

## Body format

### Short summary

One or two sentences max.

### Body

Only if needed beyond the summary: bullets for reproduction, expected vs actual, distro/backend (pacman, flatpak, etc.), and UniPack version if known.
